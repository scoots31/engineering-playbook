# Retro Log

Observations from framework sessions across all machines.
Appended automatically at session end by the Stop hook.

**Written by:** session-signal-push.sh from `.claude/retro-notes.tmp`
**Read by:** framework curator only — never read by framework skills during build sessions
**Reviewed by:** framework owner in curator sessions

**Format per entry:**
```
date | user | project | phase | trigger-type | observation
```

**Trigger types:**
- `framework-detected` — skill chain produced a notable signal (stuck, code review failed, self-correction, QA caught what self-verification missed)
- `solo-expressed` — solo expressed dissatisfaction, confusion, or a "why did that happen" moment
- `explicit-flag` — solo explicitly asked to flag something for later review

**Review separator:** `--- Reviewed YYYY-MM-DD ---`
Entries above the most recent separator have been seen by the curator. New entries accumulate below.

Do not edit this file manually. Written only by the session signal hook, reviewed only in curator sessions.
