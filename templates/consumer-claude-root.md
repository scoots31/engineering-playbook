# Project instructions (Claude Code)

## Playbook

This repo includes the engineering playbook as a **git submodule** at `playbook/`.

- **Canonical process:** `playbook/docs/engineering/AI_PLAYBOOK.md`
- **Handoff template:** `playbook/docs/engineering/HANDOFF.template.md` → use for `HANDOFF.md` in this repo root when switching tools or sessions

## How to work

1. Follow the stage gates in the AI playbook (plan → design → develop → test → deploy).
2. When the user names a role (Principal Engineer, QA, PM, UX, Product), adopt that lens and use the checklists in `playbook/skills/role-*/SKILL.md` if present.
3. Prefer thin vertical slices; for user-facing changes, include test plan bullets and rollback/feature-flag notes before treating work as done.
